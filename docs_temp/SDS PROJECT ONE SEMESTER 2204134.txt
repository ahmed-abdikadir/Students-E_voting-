1.0 INTRODUCTION
The Software Design Specification Document is a document aiming to aid in the process of developing software by providing the system details on how it will be built. Within a Software Design Document, various forms of graphical documentation are used to give a clear display of the system’s functionality. 
This document provides the details regarding the design of the Online voting system for Kcau University. Its design aims to meet the new system’s requirements while achieving a high-quality standard.
System design is concerned with creating a system with the following standards;
	•	Secure.
	•	Reliable. 
	•	Ease of interoperability. 
	•	Easy to use.


1.1 Purpose
The purpose of this document is to provide a clear description of the system design allowing for it to be implemented in software development and bringing clarity as to what the system should and will do after the design process.
 
1.2 Scope
This Software Design Specification Document will provide an overview of the system with a base-level focused view of the functionality of the system. 
This document is a complete description of the design of the Online Voting System with the basic architecture being an Application server.


1.3 System Design Constraints 
The major design constraint is the requirements specifications document, which was garnered during the system analysis phase. The requirements specification document contains all the requirements as expressed and agreed upon by the stakeholders of the system to be included in the design of the new system.



Another design constraint is the environmental constraint. 
This includes the implications of the new system on the hardware and software of the current machines. It includes;
	•	Increase in memory and storage capacity for data in terms of hardware.
	•	Interoperability of the previous and incoming system. 
	•	Database management techniques require knowledge of databases. 


1.4 Design Goals and Objectives 
The goal of the Online voting system is to develop a system which bridges the gap in functionality between the old and the new system supporting the electoral process in higher educational institution.
Software design specifically focuses on ensuring a quality system. The design ensures a stable system serving as the foundation for the development of the system. 
Objectives; 
	•	Reliability - The system design should ensure that the system achieves full reliability for the system users. 
	•	Security - The system design should ensure that access to the system is limited to the respective users.
	•	Ease of Use - System design should implement a user-friendly view of the system.

2.0 SYSTEM ARCHITEURE
 2.1 Introduction 
As the blueprint of the system, the system architecture aims to provide an overview of the system and its functionalities. This is achieved by breaking down the system into its various components and then applying the requirements and building the modules themselves. 

2.2 Client-Server model 
The client-server architecture as a distributed system model allows the server to give access to other workstations in terms of data and enhancing accessibility. 
	•	With the main server offering services to other sub-systems it creates an effective workstation by creating a system of distribution. 
	•	Clients, on the other hand, use the services offered by the server concurrently across all the workstations.
	•	A network aims to connect the clients to the server, with the clients accessing the server through remote procedure calls.

2.3 Design Approach 
Modularity is the main factor in the design approach. With separate modules, the system is more effective in the case of system failure since the whole system will not fail but the specific module hence allowing for the rest of the system to continue functioning like before.

 2.4 Architectural Design 
For the architectural design, the structure of the program comes to a mold. Defining the data flow within the system, together with the interfaces that have been developed in the program gives a clear picture of the architectural design.

 2.5 Logical Design 
The logical design of the system is concerned with the system functions. What the system can do and how it will be done are the main concerns. 

2.6 Use Case Diagram 
A use case diagram is a graphical depiction of a user’s interaction with the system.
These actors ensure the system operates effectively, securely, and transparently, maintaining trust in the voting process.


2.6.1 Actors
The actors are the users interacting with the system. The actors in this system are;
Voters
They register and verify their eligibility to vote, cast votes in the election and receive confirmation that their vote has been submitted successfully.
Administrators
They manage the voter database, monitor the voting process to ensure it runs smoothly, resolve issues related to voter access or system functionality and publish results after the election concludes.
Candidates
They represent parties, individuals, or options voters can select and they may indirectly interact with the system to verify their inclusion in the candidate list.


Auditors
They:
	•	Conduct system audits to ensure compliance with regulations and standards.
	•	Verify that votes are counted correctly and securely.
	•	Assess system security and detect potential fraud or irregularities.

System
The role of the system as an actor includes;
	•	Authenticate users and provide access controls.
	•	Facilitate the casting and recording of votes.
	•	Encrypt and secure voter data and ballots.
	•	Generate and display real-time or post-election results

3.0 DATABASE DESIGN 
3.1 Introduction
 A database is an organized collection of structured data typically stored electronically in a computer system. An effective database maintains consistency while managing access to data through the use of privileges. 
SQL acts as a standard language to be used in database management, with SQL statements, we can create interactive queries to perform operations on data. The database of this project is going to be used together with the MySQL database management system.
Reasons for using MySQL includes:
	•	Scalability: Handles large voter databases efficiently, supporting growing election data.
	•	Security: Provides robust authentication, encryption, and data access controls, crucial for sensitive voting data.
	•	Reliability: Ensures high availability and fault tolerance for uninterrupted voting processes.
	•	Performance: Optimized for fast read/write operations, ensuring quick responses during peak voting periods.
	•	Flexibility: Supports integration with web applications and diverse programming languages.
	•	Cost-Effective: Open-source and free to use, reducing overall project costs.
3.2 Normalization 
Normalization is the process of organizing data in a database. 
Normalization includes creating tables and establishing relationships between the tables according to the rules concerning data protection and eliminating redundancy and inconsistency dependency.
Normalization generally aims to correct the anomalies in the database.
